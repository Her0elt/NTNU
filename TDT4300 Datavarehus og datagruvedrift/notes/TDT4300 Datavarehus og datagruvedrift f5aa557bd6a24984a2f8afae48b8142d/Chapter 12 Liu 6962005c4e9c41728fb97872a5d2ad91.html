Web-based organizations collect vast amounts of clickstream and user data from e-commerce, web services, and web-based information systems. Analyzing this data can help organizations in various ways, such as determining the lifetime value of clients, designing cross-marketing strategies, evaluating promotional campaigns, and optimizing web-based applications. Web usage mining is an automatic discovery and analysis of patterns in clickstream and associated data collected from user interactions with one or more web sites. This process can be divided into three stages: data collection and pre-processing, pattern discovery, and pattern analysis. During data pre-processing, clickstream data is cleaned and partitioned into a set of user transactions representing their activities during different visits. The pattern discovery stage involves performing statistical, database, and machine learning operations to obtain hidden patterns reflecting typical user behavior. Finally, the discovered patterns and statistics are processed, filtered, and used as input to recommendation engines, visualization tools, and web analytics and report generation tools.

12.1 Data Collection and Pre-Processing

Data preparation is a critical step in the Web usage mining process, and it involves creating a suitable target dataset that can be used for data mining and statistical algorithms. This step is particularly important in Web usage mining due to the characteristics of clickstream data and its relationship to other related data collected from multiple sources and across multiple channels. Preparing the data can be the most time-consuming and computationally intensive step in the process, and often requires the use of special algorithms and heuristics not commonly employed in other domains. The data preparation process may involve pre-processing the original data, integrating data from multiple sources, and transforming the integrated data into a form suitable for input into specific data mining operations. The successful application of data mining techniques to Web usage data is highly dependent on the correct application of pre-processing tasks, which include data fusion and cleaning, user and session identification, and pageview identification.

12.1.1 Sources and Types of Data

Web usage mining relies on several data sources such as server log files, application server logs, site files, metadata, operational databases, application templates, and domain knowledge. In some cases, data collection can also take place from client-side or proxy-level data collection, or from external clickstream or demographic data sources. The collected data can be categorized into four primary groups, with usage data being the most important one. Usage data is the log data collected automatically by the Web and application servers that represent the fine-grained navigational behavior of visitors. Server access logs contain fields such as the time and date of the request, the IP address of the client, the resource requested, status of the request, HTTP method used, user agent, referring Web resource, and client-side cookies. Data from the server access logs is the primary source of data in Web usage mining.

Large-scale websites use multiple web or application servers to serve content to users, and data fusion merges log files from these servers, requiring synchronization across them. Data cleaning involves site-specific tasks such as removing unnecessary references to objects and fields not useful for analysis or mining. Crawler references are removed by identifying search engine crawlers or other well-behaved crawlers that abide by standard robot exclusion protocols. Identification of pageviews depends on the site structure, content, and domain knowledge. Attributes such as pageview ID, pageview type, and metadata need to be recorded for each pageview to provide a flexible framework for data mining activities. User identification requires distinguishing unique visitors without authentication mechanisms, which can be achieved using client-side cookies or IP addresses, though the latter may not be sufficient due to ISP proxy servers.

After sessionization, path completion is important to infer missing references due to client- or proxy-side caching. The site structure and referrer information are used to heuristically infer the missing references. Data integration with preprocessed clickstream data is necessary for effective pattern discovery in e-commerce applications. This includes integrating user data, product attributes and categories, and product-oriented events such as shopping cart changes, order and shipping information, and click-throughs. The integrated data is stored in a final transaction database and an e-commerce data mart, which provides pre-computed e-metrics along multiple dimensions and is used as the primary data source for OLAP and data visualization.

12.2 Data Modeling for Web Usage Mining

Usage data pre-processing results in n pageviews, P = {p1, p2, ···, pn}, and m user transactions, T = {t1, t2, ···, tm}, where each transaction ti in T is a subset of P. Each transaction is represented as an l-length sequence of ordered pairs (pti, w(pti)), where pti = pj for some j in {1, 2, ···, n} and w(pti) is the weight associated with the pageview pti, representing its significance. The weights can be determined based on the type of analysis or intended personalization framework. Weights are usually binary or can be a function of the duration of the pageview in the user's session. The last pageview's weight is usually set to the mean time duration for the page taken across all sessions in which the pageview does not occur as the last one. In practice, a normalized value of page duration is used to account for user variances. For clustering and association rule mining, where the ordering of pageviews in a transaction is irrelevant, each user transaction can be represented as a vector over the n-dimensional space. The set of all user transactions can be viewed as an m×n user-pageview matrix (transaction matrix) denoted by UPM. Unsupervised learning techniques such as clustering of transactions, item clustering, association, and sequential pattern mining can be applied to obtain patterns. These techniques can lead to the discovery of important user or visitor segments. It is also possible to integrate other sources of knowledge, such as semantic information from the content of web pages with the web usage mining process. Each pageview p can be represented as an r-dimensional feature vector, denoted by p, where r is the total number of extracted features (words or concepts) from the site in a global dictionary.

12.3 Discovery and Analysis of Web Usage Patterns

12.3.1 Session and Visitor Analysis

Web usage data is commonly analyzed by aggregating data into predetermined units such as days, sessions, visitors, or domains, and using standard statistical techniques to gain knowledge about visitor behavior. This approach is taken by most commercial Web log analysis tools and can provide useful information about frequently accessed pages, average view time, entry and exit points, and other aggregate measures. Online Analytical Processing (OLAP) provides a more integrated framework for analysis using a multidimensional data warehouse that integrates usage, content, and e-commerce data at different levels of aggregation for each dimension. OLAP tools allow changes in aggregation levels along each dimension during the analysis, and can be based on various fields available in the log files. The integration of e-commerce data in the data warehouse can enhance the ability of OLAP tools to derive important business intelligence metrics. The output from OLAP queries can also be used as input for data mining or visualization tools.

12.3.2 Cluster Analysis and Visitor Segmentation

Clustering groups similar items together. Two kinds of clusters are user and page clusters, and clustering of user records is commonly used in web analytics. Clustering can help to infer user demographics and provide personalized content. Using k-means, transactions can be grouped together based on their similarity in a multi-dimensional space. However, each transaction cluster may contain thousands of user transactions, so the ultimate goal is to analyze each segment for deriving business intelligence or for personalization. One way to create an aggregate view of each cluster is to compute the centroid of each cluster. This aggregate representation can be used for predictive modeling and recommender systems. A filtering threshold can be used to remove pageviews that appear in a very small number of vectors in a cluster.

12.3.3 Association and Correlation Analysis

Association rule discovery and statistical correlation analysis can identify groups of items or pages that are often accessed or purchased together, allowing websites to organize content efficiently or recommend related products. The Apriori algorithm is the most common approach used to discover frequent itemsets, which are groups of items that frequently occur together in many transactions. Association rules are then generated from the frequent itemsets, where support represents the probability that two itemsets occur together in a transaction and confidence represents the conditional probability that one itemset occurs in a transaction given that another itemset has occurred. The mining of association rules in web transaction data has many advantages, including improving site structure and providing product recommendations. However, sparse datasets are a challenge, and dimensionality reduction techniques and collaborative filtering can be used to alleviate this issue. Association-based recommendation algorithms typically use a sliding window over the target user's active profile or session, and efficient trie-based data structures can be used to store discovered itemsets and generate recommendations efficiently without generating all association rules from frequent itemsets.

12.3.4 Analysis of Sequential and Navigational Patterns

Sequential pattern mining finds inter-session patterns in time-ordered sets of sessions or episodes. Web marketers use this technique to predict future visit patterns and place ads targeted at user groups. Sequential patterns (SPs) capture frequently visited web page trails, while contiguous sequential patterns (CSPs) represent pairs of adjacent items that appear consecutively in transactions. The support of a sequential or contiguous sequential pattern is the fraction of transactions that contain it. CSPs capture frequent navigational paths among user trails, while SPs can represent more general navigational patterns within a site. Web transactions can be modeled as a Markov model, where each page view or category is a state and the transition probability between states represents the likelihood of a user navigating from one state to the next. Markov models are useful for predictive modeling and for discovering high probability user navigational trails in a website. Mixtures of Markov models can be used to cluster navigational sequences and explore users' navigational behavior.

12.3.5 Classification and Prediction based on Web User Transactions

Classification is the task of mapping a data item into one of several prede- fined classes. In the Web domain, one is interested in developing a profile of users belonging to a particular class or category. This requires extraction and selection of features that best describe the properties of given the class or category. Classification can be done by using supervised learning algo- rithms such as decision trees, naive Bayesian classifiers, k-nearest neighbor classifiers, and Support Vector Machines (Chap. 3). It is also480 12 Web Usage Mining possible to use previously discovered clusters and association rules for classification of new users (Sect. 3.5). Classification techniques play an important role in Web analytics appli- cations for modeling the users according to various predefined metrics. For example, given a set of user transactions, the sum of purchases made by each user within a specified period of time can be computed. A classifica- tion model can then be built based on this enriched data in order to classify users into those who have a high propensity to buy and those who do not, taking into account features such as users’ demographic attributes, as well their navigational activities. Another important application of classification and prediction in the Web domain is that of collaborative filtering. Most collaborative filtering applications in existing recommender systems use k-nearest neighbor clas- sifiers to predict user ratings or purchase propensity by measuring the cor- relations between a current (target) user’s profile (which may be a set of item ratings or a set of items visited or purchased) and past user profiles in order to find users in the database with similar characteristics or prefer- ences [236]. Many of the Web usage mining approaches discussed earlier can also be used to automatically discover user models and then apply these models to provide personalized content to an active user [386, 445]. Basically, collaborative filtering based on the k-nearest neighbor (kNN) approach involves comparing the activity record for a target user with the historical records T of other users in order to find the top k users who have similar tastes or interests. The mapping of a visitor record to its neighbor- hood could be based on similarity in ratings of items, access to similar con- tent or pages, or purchase of similar items. In most typical collaborative filtering applications, the user records or profiles are a set of ratings for a subset of items. The identified neighborhood is then used to recommend items not already accessed or purchased by the active user. 12.4 Discussion and Outlook

Web usage mining has emerged as the essential tool for realizing more personalized, user-friendly and business-optimal Web services. Advances in data pre-processing, modeling, and mining techniques, applied to the Web data, have already resulted in many successful applications in adap- tive information systems, personalization services, Web analytics tools, and content management systems. As the complexity of Web applications and user’s interaction with these applications increases, the need for intel- ligent analysis of the Web usage data will also continue to grow. Usage patterns discovered through Web usage mining are effective in capturing item-to-item and user-to-user relationships and similarities at the level of user sessions. However, without the benefit of deeper domain knowledge, such patterns provide little insight into the underlying reasons for which such items or users are grouped together. Furthermore, the in- herent and increasing heterogeneity of the Web has required Web-based applications to more effectively integrate a variety of types of data across multiple channels and from different sources. Thus, a focus on techniques and architectures for more effective integra- tion and mining of content, usage, and structure data from different sources is likely to lead to the next generation of more useful and more intelligent applications, and more sophisticated tools for Web usage mining that can derive intelligence from user transactions on the Web.
